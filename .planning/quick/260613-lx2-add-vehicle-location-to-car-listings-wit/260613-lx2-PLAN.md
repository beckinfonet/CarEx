## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile -> backend GET /api/geocode/city | Untrusted `q` query param crosses into a server-side outbound HTTP call |
| backend -> Nominatim | CarEx server calls a third-party public API on behalf of the user |
| mobile -> backend POST/PUT /api/cars | `location` string (client-normalized) crosses into persisted Car data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-lx2-01 | Tampering | server.js POST/PUT /api/cars `location` | accept | Client sends the already-normalized geocode string; backend stores it as free-text data (same trust model as existing exteriorColor). v1 does not re-verify server-side; low value, display-only string, no injection sink (rendered as RN Text). |
| T-lx2-02 | Denial of Service | GET /api/geocode/city -> Nominatim | mitigate | AbortController ~5s timeout in geocodeService; all errors/timeouts return { ok:false } (never hang/500). Endpoint is public but does only a single rate-limited-by-policy outbound call with the required User-Agent; no amplification (limit=1, single result). |
| T-lx2-03 | Information disclosure | geocodeService outbound call | accept | Only the typed city string + a static contact UA leave the server; no user PII, tokens, or secrets sent to Nominatim. |
| T-lx2-04 | Tampering (SSRF-ish) | `q` interpolated into Nominatim URL | mitigate | `q` is URL-encoded (encodeURIComponent) into a fixed Nominatim host/path; no user-controlled host or scheme — cannot be redirected to an internal target. |
</threat_model>

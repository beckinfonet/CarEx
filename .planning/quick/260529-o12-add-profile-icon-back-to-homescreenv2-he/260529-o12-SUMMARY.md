# Quick Task 260529-o12: Restore Profile Icon on HomeScreenV2 Summary

Added a V2-themed circular `ProfileAvatarButton` to the LEFT of the `FloatingSearchPill` on HomeScreenV2, mirroring v1's profile-button tap routing (Profile when logged in, Login when logged out) and avatar/icon conditional, via a new optional `leading` slot on the pill.

## What Was Built

- **`ProfileAvatarButton.tsx`** (new): A focused circular button. Calls `useNavigation` + `useAuth`; `handlePress` navigates to `Profile` when `user` is truthy, else `Login`. Renders `user.avatarUrl` as a cover `Image` when present, otherwise a lucide `User` icon (`V2.text` when logged in, `V2.textMuted` when out). Styled with `V2.surface` background, `V2.borderHi` border, `overflow: 'hidden'`, default `size=48`. No v1 COLORS imported; no new i18n keys; no new deps.
- **`FloatingSearchPill.tsx`** (modified): Added optional `leading?: React.ReactNode` to props. The `wrapper` View became a row (`flexDirection: 'row'`, `alignItems: 'center'`, `gap: 10`) with `{leading}` rendered as the first child (left of the pill). `paddingHorizontal: 18, paddingTop: 12` preserved on the wrapper (still the only place horizontal padding lives). The `pill` style gained `flex: 1` so it shrinks to make room. Pill internals (Search icon, placeholder, filter button) unchanged.
- **`HomeScreenV2.tsx`** (modified): Imported `ProfileAvatarButton`; passed `leading={<ProfileAvatarButton />}` into the existing `<FloatingSearchPill />` render site. No new wrapper/row added — the pill remains the direct child of SafeAreaView above the FlatList, preserving sticky behavior.

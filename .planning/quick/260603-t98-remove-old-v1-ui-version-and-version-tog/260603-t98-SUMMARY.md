## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Wrapped BigFeedCard/SmallFeedCard tests in LanguageProvider**
- **Found during:** Task 4 (final gate, `npm test`)
- **Issue:** `BigFeedCard` and `SmallFeedCard` render `ListingStatusBadge`, which calls `useLanguage()` unconditionally (before its `return null`). The plan's Task 3 only specified stripping the `UIVersionProvider` wrapper. With no provider, these two suites threw `useLanguage must be used within a LanguageProvider`.
- **Pre-existing evidence:** I checked out the **base-commit** versions of these two test files (wrapped only in `UIVersionProvider`, which never supplied language) and ran them in place — both failed at base with the identical `useLanguage must be used within a LanguageProvider` error (`2 failed, 2 total`). So these were already broken before this task; my wrapper-strip did not introduce the failure, it merely changed the missing-provider error from latent to surfaced. The fix makes them pass.
- **Fix:** Added `import { LanguageProvider }` and wrapped the rendered component in `<LanguageProvider>` in both test files.
- **Files modified:** `src/components/home/v2/__tests__/BigFeedCard.test.tsx`, `src/components/home/v2/__tests__/SmallFeedCard.test.tsx`
- **Verification:** `npx jest BigFeedCard.test.tsx SmallFeedCard.test.tsx` → `2 passed, 2 total`.
- **Committed in:** `40198cf`

**2. [Rule 1 - Bug] Updated useTypography test assertion to V2-hardcoded behavior**
- **Found during:** Task 3 (test wrapper strip)
- **Issue:** The existing `useTypography` test asserted the V1 contract (`display`/`mono` are `undefined`) and wrapped in `UIVersionProvider`. After the hook is V2-hardcoded, that assertion is invalid.
- **Fix:** Removed the provider wrapper and changed assertions to expect `'Manrope'` and `'JetBrainsMono-Medium'`.
- **Files modified:** `src/hooks/__tests__/useTypography.test.tsx`
- **Verification:** Passes in full suite run.
- **Committed in:** `64884b7` (Task 3 commit)

|----------|-------------|
| App-layer (handler) ã Mongoose model API | All writes to `listing_moderation_actions` MUST go through `.create()`. Mongoose's `.create()` bypasses no pre-hook because pre-save runs on insert; pre-update/pre-delete hooks block every other write path. |
| Database driver ã collection | A direct `db.listing_moderation_actions.updateOne(...)` from the MongoDB shell BYPASSES Mongoose hooks ã D-13 (no hash-chain) + D-11 acknowledge this; deferred to future security hardening per 07-CONTEXT.md ôÏDeferred Ideas. |
| Phase 8 handler trust | Phase 8 endpoint handlers will be the only legitimate writers; this plan provides the application-layer guarantee that no subsequent code path can mutate an already-written audit row. |

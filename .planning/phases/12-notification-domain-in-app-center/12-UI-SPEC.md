## Spacing Scale

Reuse `SIZES` from `theme.ts` verbatim [theme]. All values are multiples of 4.

| Token | Value | `theme.ts` key | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `spacingXs` | Icon-to-label gaps, badge inner padding |
| sm | 8px | `spacingSm` | Compact element spacing, chip gaps |
| md | 16px | `spacingMd` / `padding` | Default screen + row padding |
| lg | 24px | `spacingLg` | Section padding, settings group separation |
| xl | 32px | `spacingXl` | Major layout gaps |
| 2xl | 48px | `spacing2xl` | Empty-state vertical centering blocks |

Radii: `radiusSm` 8 (buttons, chips, WatchButton pill if not full-pill), `radiusMd` 12 (cards, feed items), `radiusPill` 999 (badges, count bubble). [theme]

**Exceptions:**
- `minTapTarget` 44px — every interactive control (WatchButton, SaveSearchBar tap, feed row, toggle, delete/undo affordance) MUST meet a 44px touch target [theme `minTapTarget`]. This is a deliberate non-4-multiple touch-accessibility floor, not a layout spacing value.
- Unread count badge bubble: ~18–22px diameter (`radiusPill`), 2px inner horizontal padding for the "9+" string — small by design [CTX D-07].

## Color

Dark theme, reused from `COLORS` [theme]. 60/30/10 split mapped to existing tokens:

| Role | Value | `theme.ts` key | Usage |
|------|-------|----------------|-------|
| Dominant (~60%) | `#0F1115` | `background` | Screen backgrounds (NotificationsScreen, NotificationSettingsScreen), BottomBar |
| Secondary (~30%) | `#181B21` | `cardBackground` | Feed item cards, settings group cards, MoreMenu sheet, toast surface |
| Accent (~10%) | `#3B82F6` | `accent` | See reserved-for list below |
| Destructive | `#EF4444` | `destructive` | Delete-subscription affordance, destructive confirm |

Supporting tokens (already in theme, used as-is):
- `textPrimary` `#FFFFFF` — unread feed title, primary labels
- `textSecondary` `#9CA3AF` — read feed items, timestamps, descriptions, settings hints
- `textTertiary` `#6B7280` — disabled "Daily — coming soon" cadence option [CTX D-10]
- `border` `#2A2F3A` — feed-item dividers, card borders, settings group separators
- `success` `#22C55E` / `successFg` `#4ADE80` — toast confirmation accent ("We'll alert you…")
- `warning` `#F59E0B` — reserved; not required this phase

**Accent (`#3B82F6`) reserved for — explicit list (never "all interactive elements"):**
1. The **unread-count badge** ("9+") on the MoreMenu Notifications item [CTX D-07]
2. The **red dot** on the BottomBar More button — uses `destructive` red `#EF4444`, NOT accent (matches "red dot" wording in [CTX D-07]; this is a deliberate exception so the dot reads as an alert signal, distinct from accent-blue chrome)
3. The **WatchButton** bell pill (active/inactive states: accent border + accent icon when watchable; accent fill `rgba(59,130,246,0.1)` when actively watching, mirroring `BottomBar.activeButton`) [CTX D-01]
4. The **SaveSearchBar** CTA emphasis (accent text/border) [CTX D-08]
5. **Unread feed-item left indicator** — a thin accent left-border or accent dot marking unread vs read [CTX D-15, NCEN-04]
6. **Selected cadence** option ("Instant") in the cadence selector [CTX D-10]

**Watch vs Favorite disambiguation (load-bearing):** The Watch control is accent-blue + a `Bell` icon + a **text label**; the existing Favorite is an icon-only **red** `Heart`. Color (blue vs red) + label vs icon-only + shape (pill vs bare icon) all differ — three independent disambiguators per [CTX D-01]. The red dot on More (`destructive` red) is the ONLY other red signal and lives in a different surface (BottomBar), so no Watch/Favorite/alert collision.

**Primary CTA for this phase:** "Сообщать о новых вариантах" / "Notify me about new matches" (the save-search conversion moment, sticky bar) [CTX D-08].

**Empty state** ([CTX D-14], NCEN-05): heading + body above, `BellOff` icon, centered. No CTA button — the guidance is instructional (where to create subscriptions), since creation happens on CarDetails / search results, not here.

**Error states** ([REQ — inferred]): feed-load failure shows the pull-to-refresh error line (non-blocking, feed stays mounted, NPRF-07 fallback never dead-ends). Subscription create/delete failure shows the action-error line via toast/Alert and rolls back optimistic UI.

**Destructive actions:**

| Action | Confirmation approach | Copy |
|--------|----------------------|------|
| Delete a saved search | Inline `Alert.alert` confirm (existing RN pattern) before delete; row removed on confirm [CTX D-11] | Title "Удалить поиск?" / "Delete saved search?"; body "Перестанем сообщать о новых вариантах." / "We'll stop alerting you about new matches."; confirm "Удалить"/"Delete" (destructive red), cancel "Отмена"/"Cancel" |
| Delete a watched car | Inline `Alert.alert` confirm before delete [CTX D-11] | Title "Перестать отслеживать?" / "Stop watching this car?"; body "Уведомления по этому авто прекратятся." / "You'll stop getting alerts for this car."; confirm "Перестать"/"Stop" (destructive red), cancel "Отмена"/"Cancel" |
| Master mute (mute all) | No confirm — reversible toggle, not destructive [CTX D-11] | n/a |
| Save-search Undo | NOT a confirm — toast-level reversal of a just-created subscription [CTX D-09] | "Отменить"/"Undo" within the success toast |

Note: there are no hard-delete-data destructive actions in Phase 12 beyond removing the user's own subscription rows; everything is reversible via re-create. Confirms above are friction guards, not audit-grade gates.

## Interaction Contract (phase-specific behaviors)

| Behavior | Contract | Source |
|----------|----------|--------|
| Watch tap | One tap creates the subscription opting into all 4 events; button flips to "Watching" active state; no event-picker sheet | CTX D-03 |
| Watch key | Subscription keys on `car._id \|\| car.id \|\| carId` — never bare `car.id` | CTX D-04, memory |
| Save-search tap | One tap creates Saved Search from current `activeFilters` at **instant** cadence, then shows success toast with Undo; no confirm sheet | CTX D-08/D-09 |
| Save-search bar visibility | Visible only when filters are active in `SearchResultsV2` header region | CTX D-08 |
| Cadence selector | "Instant" selected (accent); "Daily" rendered greyed/disabled with "coming soon" hint; Daily not selectable in Phase 12 | CTX D-10 |
| Unread badge | Red dot on BottomBar More when unread > 0; "9+"-capped count on MoreMenu Notifications item; both derive from `NotificationContext` unread count | CTX D-07 |
| Feed | Reverse-chronological, cursor-paginated (base64 `{createdAt,_id}`); pull-to-refresh; tap deep-links via existing `linking` config | NCEN-02/03, CTX D-15 |
| Read state | Tapping a notification marks it read; "Mark all read" action in feed header; unread visually distinct via accent left indicator + 600-weight title, read items use `textSecondary` | NCEN-04, CTX D-15 |
| Push-denied fallback | In-app center remains fully functional with `fcm.send` as a no-op stub — no dead-end | NPRF-07, REQ |
| Toast component | Researcher discretion (D-13): use a lightweight in-app toast (custom or existing Alert fallback) anchored above BottomBar; must not block navigation and must auto-dismiss (~4s) with the Undo tappable for its duration | CTX D-13 / DEF |
| Quiet-hours defaults (plumbing) | Default 22:00–08:00 local (seeded from device-timezone→city signal; no GPS/per-user TZ); controls present but delivery deferred to Phase 14 | CTX D-16 / DEF |
| Daily-cap default (plumbing) | Default soft cap 3/day for instant saved-search push; control present, enforcement deferred to Phase 14 | CTX D-16 / DEF, NPRF-04 |

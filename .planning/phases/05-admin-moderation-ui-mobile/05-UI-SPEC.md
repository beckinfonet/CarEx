# Phase 5 — UI Design Contract

> Visual and interaction contract for the Admin Moderation UI on mobile (React Native). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Every decision below is lockable: the planner and executor consume this file as the visual source of truth and must not re-decide these values without explicit user override.

---

## Scope

This UI-SPEC governs:

1. `AdminManagementScreen` (repurposed per D-03) — approved-users roster with inline severity badge + trailing quick-action icon.
2. `AdminModerationScreen` (new) — search + filter + infinite-scroll user list.
3. `AdminUserDetailScreen` (new) — sticky summary card + flat moderation-history list + contextual Unsuspend CTA.
4. `ModerationActionModal` (new shared component) — single generic bottom-sheet modal driving all 5 admin actions.
5. `TypedConfirmationModal` (new shared component) — destructive-action confirmation gate.
6. Quick-action bottom sheet on AdminManagementScreen (the menu opened by the trailing icon).

**Theme reality check:** CarEx is a **dark theme** marketplace (`COLORS.background = '#0F1115'`, white text). The scope note assumption that it is "light-theme only" was incorrect. Phase 5 inherits the existing dark theme; no light-mode forks. The severity palette below is tuned for ≥4.5:1 contrast against `COLORS.cardBackground` (`#181B21`).

**No new UI libraries.** React Native 0.83 + lucide-react-native + react-native-safe-area-context + stock RN primitives (`Modal`, `FlatList`, `RefreshControl`, `TextInput`, `TouchableOpacity`, `ScrollView`, `Pressable`, `KeyboardAvoidingView`, `Animated`). This is enforced by CLAUDE.md.

## Spacing Scale

Phase 5 introduces a formal 4-point scale. Existing code uses `SIZES.padding = 16` as the baseline; Phase 5 extends `SIZES` rather than replacing it.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside chips/badges, vertical gaps inside a badge pill |
| sm | 8px | Gap between stacked meta rows inside a history card; icon-to-label in action-sheet rows |
| md | 16px | Default screen horizontal padding; card padding; gap between list items (existing `SIZES.padding`) |
| lg | 24px | Section separation inside the sticky summary card; header-to-list gap on AdminUserDetailScreen |
| xl | 32px | Vertical spacing above a destructive confirmation warning; empty-state illustration-to-text gap |
| 2xl | 48px | Empty-state container vertical padding; big-gap between header and first content block |
| 3xl | 64px | Not used in Phase 5 (reserved for future full-page empty states) |

**Exceptions (touch-target ergonomics):**
- **44px minimum tap target height** on all interactive elements (`TouchableOpacity`, row trailing action icon, chip, modal buttons). Row trailing icon uses `padding: 12` inside a 44×44 touch zone even though the glyph itself is 20px.
- **Bottom-sheet handle** (grab affordance at top of quick-action sheet) is a 36×4 pill centered with 8px vertical breathing room — standard iOS/Android bottom-sheet convention.

**Extend `src/constants/theme.ts` `SIZES` with:**
```ts
export const SIZES = {
  padding: 16,            // existing (md)
  borderRadius: 12,       // existing
  iconSize: 24,           // existing
  // --- Phase 5 additions ---
  spacingXs: 4,
  spacingSm: 8,
  spacingMd: 16,          // alias for padding; use spacingMd in new code
  spacingLg: 24,
  spacingXl: 32,
  spacing2xl: 48,
  radiusSm: 8,            // chips, inputs, buttons
  radiusMd: 12,           // alias for borderRadius; cards
  radiusPill: 999,        // badges
  minTapTarget: 44,       // minimum touch zone
  badgeHeight: 22,        // severity badge height
  chipHeight: 32,         // filter chip height
  bottomSheetHandleWidth: 36,
  bottomSheetHandleHeight: 4,
};
```

## Typography

React Native system font. Phase 5 declares **4 sizes × 2 weights**. All new `<Text>` styles on Phase 5 screens reference these tokens.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 20px (ratio 1.43) | Meta rows (admin email, timestamps), card captions, note text (italic variant), chip labels |
| Label | 16px | 400 (regular) | 24px (ratio 1.5) | Primary row label (user email in list rows), form input text, modal body copy |
| Heading | 20px | 600 (semibold) | 26px (ratio 1.3) | Screen titles inside header bars, modal titles, summary card user email |
| Display | 28px | 600 (semibold) | 34px (ratio 1.21) | Not used in Phase 5 (reserved for future error-state / empty-state illustration headings if added) |

**Token-level additions (append to `COLORS` / or create `TYPOGRAPHY` object — planner's call during Plan 05-01):**
```ts
export const TYPOGRAPHY = {
  body:    { fontSize: 14, fontWeight: '400', lineHeight: 20 },
  bodyStrong: { fontSize: 14, fontWeight: '600', lineHeight: 20 }, // badge labels
  label:   { fontSize: 16, fontWeight: '400', lineHeight: 24 },
  labelStrong: { fontSize: 16, fontWeight: '600', lineHeight: 24 }, // primary row text
  heading: { fontSize: 20, fontWeight: '600', lineHeight: 26 },
  display: { fontSize: 28, fontWeight: '600', lineHeight: 34 },
} as const;
```

**Italics:** Only `note` free-text inside history cards uses italic (`fontStyle: 'italic'`). Nothing else. Rationale: note is direct-quote admin content; italic visually separates it from structured meta.

**Numeric tabular:** Timestamps (`YYYY-MM-DD HH:mm`) use the default system font; no `fontVariant: ['tabular-nums']` required at 14px in the history list because rows are short and right-aligned timestamps aren't in Phase 5's layout (absolute times live in the meta row, left-aligned).

### Core dominant / secondary / accent (inherited — unchanged)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F1115` (`COLORS.background`) | Screen background on all 3 screens + modal overlay base |
| Secondary (30%) | `#181B21` (`COLORS.cardBackground`) | User list rows, history cards, summary card, modal content surface, bottom sheet |
| Accent (10%) | `#3B82F6` (`COLORS.accent`) | **Reserved for:** (1) primary CTAs (modal Confirm button, sticky Unsuspend button), (2) active filter chip background/border, (3) focused search input border, (4) ActivityIndicator tint, (5) RefreshControl `tintColor`, (6) the selected severity-picker radio card's border. Nothing else. |
| Destructive | `#EF4444` | Destructive CTAs (TypedConfirmationModal Confirm button, destructive row action icons like Trash2). Also used as the permanent-ban accent glyph color. |

**Accent is not a general "primary color."** Inline links, headings, inactive chips, badges, and icons do NOT use accent. Admin identity "Super Admin" ribbon uses its own amber (`#F59E0B`) already established in `AdminManagementScreen`.

### Severity palette (NEW — Phase 5 introduces)

Severity colors must be **defined once** and reused by Phase 6 `UserStatusBanner` verbatim (per CONTEXT.md Specifics note). Extend `COLORS` with a nested `moderation` object so Phase 6 imports `COLORS.moderation.featureLimited` directly:

| State | Key | Background (pill) | Foreground (text + icon) | Border | Ratio vs `#181B21` |
|-------|-----|------------------|--------------------------|--------|---------------------|
| `active` | `moderation.active` | `rgba(34, 197, 94, 0.15)` | `#4ADE80` (green-400) | `#4ADE80` | 7.8:1 ✓ |
| `feature_limited` | `moderation.featureLimited` | `rgba(245, 158, 11, 0.15)` | `#FBBF24` (amber-400) | `#FBBF24` | 9.1:1 ✓ |
| `blocked_with_review` | `moderation.blockedReview` | `rgba(239, 68, 68, 0.15)` | `#F87171` (red-400) | `#F87171` | 5.7:1 ✓ |
| `permanently_banned` | `moderation.permaBanned` | `rgba(156, 163, 175, 0.15)` | `#9CA3AF` (neutral-400) | `#6B7280` | 6.4:1 ✓ |

**Final TS shape to add to `theme.ts`:**
```ts
export const COLORS = {
  // ... existing ...
  moderation: {
    active:          { bg: 'rgba(34, 197, 94, 0.15)',   fg: '#4ADE80', border: '#4ADE80' },
    featureLimited:  { bg: 'rgba(245, 158, 11, 0.15)',  fg: '#FBBF24', border: '#FBBF24' },
    blockedReview:   { bg: 'rgba(239, 68, 68, 0.15)',   fg: '#F87171', border: '#F87171' },
    permaBanned:     { bg: 'rgba(156, 163, 175, 0.15)', fg: '#9CA3AF', border: '#6B7280' },
  },
  destructive: '#EF4444',   // explicit token (was ad-hoc before)
  warning:     '#F59E0B',   // amber — superadmin, destructive warnings in modals
  successFg:   '#4ADE80',   // success toasts / confirmation glyph (aligns with moderation.active.fg)
};
```

**Reason-category chip colors** (used in history cards + in the reason picker): single subtle neutral fill, not per-category color.
- Background: `rgba(156, 163, 175, 0.15)` (neutral-400 at 15% opacity)
- Foreground: `#D1D5DB` (neutral-300) — reuse `COLORS.textSecondary` is too dim; define `COLORS.textTertiaryStrong = '#D1D5DB'` or use existing `#9CA3AF` if contrast holds. Default: use `#D1D5DB` explicitly.

**Role badge colors** (on user list rows — shown alongside severity badge):
- `admin` / `superadmin` → amber `#F59E0B` + amber-15% bg (reuses existing convention from `AdminManagementScreen`)
- `broker` → accent blue `#3B82F6` + accent-15% bg
- `seller` → cyan `#06B6D4` + cyan-15% bg  *(new token)*
- `logistics` → violet `#A78BFA` + violet-15% bg  *(new token)*
- `buyer` (no provider role) → **no role badge** (default state; user is just a buyer)

Role badges are flat filled pills with 10–11px text. Multi-role users render multiple stacked (max 2 horizontally; overflow with "+N").

**Submission flow (D-08 — optimistic):**
1. User taps Confirm.
2. Modal sets `submitting: true`, button shows ActivityIndicator, all fields disabled.
3. Modal calls `props.onSubmit(payload)` — parent (row or detail screen) does:
   a. Close modal immediately.
   b. Flip the target user's badge / state locally in list/card.
   c. Fire `ModerationService.{action}(targetUid, body)`.
   d. On success: do nothing visual (optimistic state already correct). Optional Alert on detail screen only.
   e. On error (axios error OR ModerationError): revert the optimistic flip, `Alert.alert(t.error, mappedErrorMessage)`.
4. Modal closes on step 3a regardless of outcome.

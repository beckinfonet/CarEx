    **Change 4 — Replace `refreshUser` (current lines 61-68) with a deduped + cooldown-enforced version + a separate internal path:**
    ```typescript
    // Internal refresh — used by both public refreshUser AND the interceptor listener.
    // `skipInterceptor` controls whether the HTTP request bypasses the 403 interceptor.
    const refreshUserInternal = async ({ skipInterceptor }: { skipInterceptor: boolean }) => {
      if (!user?.localId) return; // D-16: logged-out skip

      if (refreshInFlightRef.current) {
        return refreshInFlightRef.current; // D-15: dedupe
      }

      // Shared 30s cooldown between AppState + 403 interceptor paths (04-CONTEXT specifics line 246).
      const now = Date.now();
      if (now - lastRefreshAtRef.current < 30_000) {
        return; // within cooldown — silent no-op
      }

      refreshInFlightRef.current = (async () => {
        try {
          const config = skipInterceptor ? { _skipModerationInterceptor: true } : undefined;
          const backendUser = await AuthService.getBackendUser(user.localId, config);
          if (backendUser) {
            const updatedUser = { ...user, ...backendUser };
            setUser(updatedUser);
            await checkAdminStatus(user.localId);
          }
          lastRefreshAtRef.current = Date.now();
        } finally {
          refreshInFlightRef.current = null;
        }
      })();

      return refreshInFlightRef.current;
    };

    // Public refreshUser — used by AppState hook, existing call sites, and screens.
    // Default path does NOT skip the interceptor (so a 403 during a manual refresh still triggers
    // the banner-update cycle via the refresh listener).
    const refreshUser = async () => {
      await refreshUserInternal({ skipInterceptor: false });
    };
    ```

    Why two paths (`refreshUser` vs `refreshUserInternal`):
    - When the 403 response interceptor calls `refreshUser` via the listener, it MUST set `_skipModerationInterceptor: true` to prevent infinite loop (D-11).
    - When AppState or a screen calls `refreshUser` directly, we don't need the flag — if the refresh ITSELF 403s, the interceptor catches it and calls the listener which sets the flag. Normal cycle, no loop.
    - Single shared implementation via `refreshUserInternal` with a boolean param keeps the logic in one place.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Denial of Service | interceptor recursion — 403 during refresh re-triggers interceptor | mitigate | `refreshUserInternal({ skipInterceptor: true })` from the listener sets `_skipModerationInterceptor: true` on the axios config. Plan 04-01's interceptor skips when flag is set. Acceptance: `grep -rc _skipModerationInterceptor src/` shows exactly 2 files. |
| T-04-04-02 | Denial of Service | AppState + 403 both trigger refresh in same tick → double fetch | mitigate | Module-level `refreshInFlightRef` dedupes — second caller awaits the first's promise (D-15). Test 3 verifies exactly 1 `getBackendUser` call for 2 parallel refreshUser invocations. |
| T-04-04-03 | Denial of Service | AppState refresh spam on rapid foreground cycles | mitigate | 30s shared cooldown via `lastRefreshAtRef`. Second refreshUser within 30s returns silently (no fetch). Test 4 verifies 1 fetch at T=0+10s, 2 fetches at T=0+35s. |
| T-04-04-04 | Spoofing | stale idToken attached after logout | mitigate | `logout()` clears `currentIdTokenRef.current = null` FIRST, before any other teardown. Interceptor `if (token) { ... }` then skips the Authorization header. Test 7 asserts getter returns null post-logout. |
| T-04-04-05 | Information Disclosure | refresh during logout race → partial state leaks into re-render | accept | React state updates after unmount are no-ops (React swallows them with a warning). The `refreshInFlightRef = null` in logout defensively clears the promise pointer. Edge case, bounded impact. |
| T-04-04-06 | Elevation of Privilege | refreshUserInternal fails to wait for dedupe before setState | mitigate | refreshInFlightRef check happens BEFORE the `const now = Date.now()` cooldown check. Parallel callers share the same promise; no second inFlight ever created. |
| T-04-04-07 | Repudiation | interceptor listener swallows refresh errors silently | mitigate | Plan 04-01's interceptor wraps the listener call in try/catch with `console.error`. This plan's listener body does NOT swallow — errors bubble up into the interceptor's handler which logs. |

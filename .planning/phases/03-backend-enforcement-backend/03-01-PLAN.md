carSchema.pre(/^find/, async function () {
  if (this.getOptions().includeAllUsers) return;
  const User = mongoose.model('User');          // lazy resolve, avoid cycle
  const hiddenUids = await User.distinct('firebaseUid', {

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public read query → Mongo | Untrusted pagination/filter params flow into `Car.find`/`Broker.find`/`LogisticsPartner.find`. The pre-hook must NOT let caller-supplied query options override `includeAllUsers`. |
| admin read path → Mongo | Admin handlers explicitly opt out via `.setOptions({ includeAllUsers: true })`. If a caller-supplied body value named `includeAllUsers` could reach `setOptions`, it becomes an authz bypass. |
| lazy mongoose.model('User') → User collection | Resolution happens per-query; if User model fails to register (e.g., test that forgets to `require` it first), the hook throws and crashes the query. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | I (Info Disclosure) | Car/Broker/LogisticsPartner pre-hook | mitigate | Default is hide-safely (no flag = filter applies). `includeAllUsers` is an explicit boolean option, never reflected from request body. Plan 03-03 wires admin routes to set it explicitly; no middleware reads it from `req.body`. |
| T-03-01-02 | E (Elevation / authz bypass) | `includeAllUsers` flag | mitigate | Bypass lives at `query.options`, not `query.filter`. Mongoose does not populate `options` from filter literals — so a user passing `{ includeAllUsers: true }` in a filter does NOT set the option. Plan 03-06 acceptance test asserts filter-literal cannot bypass. |
| T-03-01-03 | D (DoS) | `User.distinct` on every find | accept | Phase 1 D-13 pinned `{ 'moderationStatus.state': 1 }` index on User. The distinct is index-backed at expected scale. Denylist-cache strategy deferred per CONTEXT.md §Deferred Ideas. |
| T-03-01-04 | T (Tampering) | Lazy `mongoose.model('User')` | accept | If `User.js` is not loaded before the first Car/Broker/Logistics find, `mongoose.model('User')` throws `MissingSchemaError`. This is fail-loud, not silent data leak. Acceptable — crash is safer than filter-bypass. |
| T-03-01-05 | I (Info Disclosure) | Unsuspend round-trip | mitigate | Round-trip test in Plan 03-06 (hideOnFind.test.js) asserts doc fields are unchanged in DB across suspend/unsuspend — proves visibility is purely read-time (ENF-04). |
</threat_model>

## What Shipped

Admins can act on bad-actor users after they're already in the system — without losing the audit trail or breaking in-flight orders for legitimate counterparties. Six phases delivered backend-first: schema + auth → admin endpoints → enforcement → mobile plumbing → admin UI → affected-user UX + security review. The backend repo (`backend-services/carEx-services`) was modified in phases 1–3 + the 0a/0b read endpoints in phase 5; the mobile repo (`carEx`) was modified in phases 4–6.

## Cross-Phase Bug Fixes Shipped Alongside (2026-04-29 / 2026-04-30)

Three production bug fixes shipped during the milestone close window:
1. **Android deep link** — `assetlinks.json` certificate fingerprints fixed via Railway env var update on `carEx-services`. Shared SMS listing links now open the app directly via App Links.
2. **No install prompt for shared listings without app** — Server-side User-Agent redirects in Vercel (`carex-web/vercel.json`) + new `Listing.js` page + `apple-itunes-app` smart-app-banner meta tag. Mobile users without the app now auto-redirect to App Store / Play Store.
3. **Listing images blanking on Android** — Retry-on-error with cache-busting URL in `OptimizedImage` (defeats Glide's negative cache) + picker-time shrinkage (`maxWidth: 2048, quality: 0.8`). Verified live on TestFlight + Play internal.

## Milestone Decisions (verbatim from STATE.md)

- **D-15** (Phase 03-02): 403 account_suspended response body sends `status: state` (string), NOT the whole moderationStatus subdoc — mobile banner matches on the string per D-15 and avoids leaking `setByAdminUid` to gated users.
- **D-13** (Phase 03-06): Concurrent-race tests use Promise.allSettled + branch classification on confirmResult.status; exactly two valid outcomes enumerated (refund-abort OR booking-succeed), forbidden third state (booked orders + refund fired) fails the test.
- **D-16** (Phase 05): Two backend read routes (GET /api/admin/moderation/:targetUid/history + GET /api/admin/users/search) locked at the planning level. Cursor format: opaque base64 of `(createdAt, _id)` for sort-stable pagination.
- **D-08** (Phase 06-02): Single Alert button `appealOk` handles mailto-fallback UX. Struck keys (appealCopyEmail / appealCopied / appealCancel) deliberately absent — RN 0.83 removed legacy @react-native-community/clipboard.
- **D-22** (Phase 05-06): Dual-role delete contract resolved at the UI layer. QuickActionSheet predicate `hasBroker && hasLogistics` renders TWO distinct rows (deleteBrokerProfile + deleteLogisticsProfile), each with explicit `role` payload.
- **Operator deferral (2026-04-19)**: 06-0a + 06-0b QUAL-02 load test deferred indefinitely; not blocking merge per 06-SECURITY.md Section (e).
